\section{Experimental Setup}
\label{sec:exp_setup}

All experiments were conducted using the pipeline described in Chapter~\ref{chap:methodology}. We evaluate four modes---\texttt{kmeans}, \texttt{deccs}, \texttt{ddc}, and \texttt{ddeccs}---on two datasets: AwA2 (50 classes, 85 attributes, 29857 training instances) and aPY (evaluated with both 32 classes and the DDC-matching 15-class subset). Following DDC's protocol, each experiment is repeated 10 times with different random seeds, and we report mean $\pm$ standard deviation.

Experiments were run on an NVIDIA GeForce RTX 4090 GPU. Feature extraction takes approximately 90 seconds per dataset; subsequent clustering and evaluation take seconds for K-means and 5--20 minutes for DECCS consensus (depending on dataset size). Features are cached to disk after the first extraction, so repeated runs skip this step entirely.

\section{Datasets}
\label{sec:datasets}

We evaluate on two established attribute-annotated image datasets used in the DDC literature.

\subsection{Animals with Attributes 2 (AwA2)}

AwA2~\citep{Xian2018} contains 37,322 images across 50 animal classes, each annotated with 85 continuous semantic attributes (e.g., \textit{furry}, \textit{stripes}, \textit{swims}). Attributes are defined per class---all instances of a given class share the same attribute vector. We use the standard 80/20 train/test split, yielding 29857 training instances.

We note that DDC's original evaluation uses the AwA1 dataset~\citep{Lampert2013} (19832 images), which has been withdrawn due to copyright restrictions. AwA2 is the recommended drop-in replacement, sharing the same class definitions and attribute structure but containing different images. This means direct numerical comparison with DDC's reported results on AwA1 is approximate.

% Source: data/AwA2-data/Animals_with_Attributes2/

\subsection{Attribute Pascal and Yahoo (aPY)}

aPY~\citep{Farhadi2009} contains 15339 object instances across 32 classes from Pascal VOC 2008 and Yahoo images, annotated with 64 binary semantic attributes (e.g., \textit{Wheel}, \textit{Furry}, \textit{Metal}). Unlike AwA2, attributes in aPY are annotated per instance---each object has its own binary attribute vector based on its bounding box crop. Our preprocessing averages these per-instance annotations into a per-class mean attribute vector, so that both datasets enter the pipeline in the same form: one continuous attribute vector per class. This discards within-class attribute variation, and Section~\ref{sec:limitations} returns to it as a limitation.

DDC reports results on a 15-class subset of aPY consisting of 5,274 instances. In our dataset, filtering to these 15 classes yields 6,487 instances; after our standard 80/20 train/test split, we cluster on 5,189 training instances. The discrepancy with DDC's reported count likely reflects differences in preprocessing or filtering criteria. Following their setup, we evaluate with both the full 32-class dataset and the 15-class subset matching DDC's configuration. The 15 classes, identified from DDC's Figure~3 ontology, are: dog, cat, monkey, zebra, bird, bag, mug, bottle, chair, car, building, bicycle, boat, sofa, and dining table.

\section{Evaluation and Interpretability Metrics}
\label{sec:metrics}

We assess results along two axes: how well clusters recover the ground-truth classes, and how interpretable the generated descriptions are. The former is captured by four standard clustering metrics (Section~\ref{subsec:clustering_metrics}); the latter by the Tag Coverage and Inverse Tag Frequency metrics introduced by DDC (Section~\ref{subsec:interp_metrics}).

\subsection{Clustering Metrics}
\label{subsec:clustering_metrics}

We evaluate clustering quality using four standard metrics:

\textbf{Normalized Mutual Information (NMI)}~\citep{Strehl2002} measures the agreement between predicted clusters and ground-truth classes, normalized to $[0,1]$. NMI $= 1$ indicates perfect agreement.

\textbf{Clustering Accuracy (ACC)} uses the Hungarian algorithm~\citep{Kuhn1955} to find the optimal one-to-one mapping between predicted clusters and true classes, then computes the fraction of correctly assigned instances.

\textbf{Adjusted Rand Index (ARI)}~\citep{Hubert1985} measures pairwise agreement between clusterings, adjusted for chance. ARI $= 0$ indicates random clustering; ARI $= 1$ indicates perfect agreement.

\textbf{Silhouette Score}~\citep{Rousseeuw1987} measures how similar instances are to their own cluster compared to other clusters, ranging from $-1$ to $+1$. Unlike the other three, it requires no ground-truth labels. Because it is quadratic in the number of instances, we compute it on a random subsample of at most 10{,}000 instances per run; this affects AwA2 and the 32-class aPY set.

\subsection{Interpretability Metrics}
\label{subsec:interp_metrics}

Following DDC, we evaluate cluster descriptions using Tag Coverage (TC) and Inverse Tag Frequency (ITF). For a cluster $C_i$ with description $D_i$:

\textbf{Tag Coverage} (DDC Eq.\ 8) is the average fraction of cluster members possessing each descriptive attribute:
\begin{equation}
    TC(C_i) = \frac{1}{|D_i|} \sum_{d \in D_i} \frac{|\{(x,t) \in C_i : d \in t\}|}{|C_i|}
\end{equation}

\textbf{Inverse Tag Frequency} (DDC Eq.\ 9) measures how unique each cluster's description is:
\begin{equation}
    ITF(C_i) = \frac{1}{|D_i|} \sum_{d \in D_i} \log \frac{K}{\sum_{j=1}^{K} \mathbb{1}[d \in D_j]}
\end{equation}

where $\log$ denotes the natural logarithm.

Higher TC indicates that the selected attributes are prevalent within the cluster; higher ITF indicates that the attributes are distinctive to that cluster rather than shared across many clusters.

We do not reimplement the full original DDC. \paragraph{A global caveat.} No row in Table~\ref{tab:ddc_comparison} compares methods on identical data, and we do not reimplement DDC. On AwA, DDC uses the withdrawn AwA1 (19{,}832 instances, $K=40$) whereas we use AwA2 (29{,}857 instances, $K=50$). On aPY, the same 15 classes yield 6{,}487 instances in our data versus DDC's reported 5{,}274 (about 23\% more); we cluster on our 5{,}189-instance training split while DDC clusters on its full set, so the underlying populations differ even though the per-set sizes are similar. Every DDC figure is taken from the published paper, not from a controlled re-run on our data and splits. The comparisons below should therefore be read as indicative of relative standing rather than exact head-to-head measurements.

Two substantive findings emerge from the comparison, read subject to the caveat above. First, our K-means baseline exceeds DDC's reported K-means baseline on both datasets (+12.7 NMI points on AwA, +2.2 on aPY)---though on AwA this is inflated by our larger $K$ (50 vs.\ 40) and different images. Second, the original DDC's trained encoder achieves higher NMI than our frozen-features approach, substantially so on aPY (+18.6 points), reflecting the benefit of refining features for the clustering task using pairwise constraints derived from semantic attributes.

\paragraph{Note on DDC's tag annotation ratio ($r\%$).} DDC's results in Table~\ref{tab:ddc_comparison} are reported at tag annotation ratio $r = 50\%$, meaning half of each instance's attributes are randomly masked during training. This masking is part of DDC's \emph{training procedure}---it simulates incomplete annotation and creates diversity in pairwise constraint generation. At lower masking rates ($r = 10\%$), DDC reports lower performance (NMI = 75.62 on AwA). DDC does not report results at $r = 0\%$, so we cannot determine how much the masking itself contributes. Our pipeline does not use masking, since we do not train with pairwise constraints; we use full (unmasked) attributes only for the post-hoc ILP description step, which computes cluster-level attribute statistics from the attribute matrix.

\section{ILP Cluster Descriptions}
\label{sec:ilp_results}

A key contribution of our approach is the generation of interpretable cluster descriptions. We focus on the ILP-optimized descriptions, which select attributes that are both prevalent within a cluster (coverage constraint) and distinctive to it (orthogonality constraint). This contrasts with a naive top-5 ranking by mean attribute value, which would surface dataset-wide attributes such as ``Occluded'' that appear in many clusters; the ILP instead skips such non-distinctive attributes in favour of ones like ``Pedal'' and ``Handlebars'' that uniquely identify a cluster.

\paragraph{These are our modes, not the DDC paper.} The descriptions in this section are generated by our pipeline (the \texttt{ddc} and \texttt{ddeccs} modes), not taken from the original DDC paper. The original DDC publishes per-cluster TC/ITF only for a controlled 10-class AwA subset (where it reports TC up to 1.0), and does not report per-dataset average TC/ITF for its full AwA or aPY runs. A fully controlled TC/ITF comparison with the original DDC would require reimplementing it, which was outside the scope of this thesis; Section~\ref{subsec:ddc_tcitf_comparison} reports the limited comparison that DDC's published figures allow. We can more directly compare our two description modes, which differ only in the clustering they describe.

\section{The Path to the Current Approach: Early Experiments}
\label{sec:early_experiments}

Before arriving at the current frozen-features pipeline, we explored several alternative approaches. We document these experiments because their outcomes directly motivated the final design.

\subsection{Autoencoder-Based Approach}

Our initial design followed the DECCS architecture more closely: a convolutional autoencoder with a tag-prediction branch, trained with a combined loss $\mathcal{L} = \mathcal{L}_{recon} + \lambda_{tag} \cdot \mathcal{L}_{tag} + \lambda_{cons} \cdot \mathcal{L}_{consensus}$. Three encoder architectures were tested: a 4-layer CNN ($128 \times 128$), a 5-layer CNN with BatchNorm ($224 \times 224$), and a frozen ResNet-101 backbone with a trainable projector.

On a 160-image sample this achieved NMI = 0.642, but full-scale evaluation on the entire 37,322-image dataset produced NMI = 0.012---essentially random clustering. These early runs predate the standardized 80/20 split adopted for the main results, which is why they are evaluated on the full image set rather than the 29{,}857-instance training split; the gap to K-means (itself NMI = 0.869 on the training split) is far too large for this difference to explain. Debugging revealed several contributing factors: DBSCAN in the ensemble was labelling all points as noise (corrupting the consensus matrix), visualization metrics were computed on different label sets than training, and the reconstruction loss on ImageNet-normalized images dominated the gradient due to a normalization mismatch between encoder input and decoder output.

\subsection{DDC-Style MLP Training}

We then implemented DDC's actual training objectives: mutual information maximization with pairwise constraints from semantic tags, using a 3-layer MLP (2048 $\to$ 1200 $\to$ 1200 $\to$ K) on frozen ResNet features. This faced a critical scaling challenge: on the full dataset the MI loss dominated from the first epoch (reaching $L_{MI} = -0.92$ after a single epoch of 116 mini-batches), locking the model into approximately 24 coarse clusters before the pairwise constraints could take effect.

Several fixes were attempted. Per-instance tag masking ($r = 0.5$) proved essential for making pairwise constraints meaningful---without it, all instances of a class shared identical attribute vectors and the pairwise selection ignored tag information. This improved results on a 1,600-sample subset (NMI = 0.124, 30 active clusters, first positive silhouette), but the improvement did not transfer to the full dataset. Replacing an initial greedy tag-selection heuristic (which selected 75 of 85 tags, providing almost no filtering) with a proper PuLP CBC solver did not resolve the MI dominance at scale either.

\subsection{The K-Means Baseline Finding}

The decisive moment came when we evaluated K-means directly on the raw 2048-dimensional ResNet features: NMI = 0.869 on AwA2---higher than any result from the trained MLP. This showed that the pretrained features already contained sufficient class-discriminative information, and that our training was destroying rather than refining it. This finding redirected the work toward the current approach: leveraging the strong pretrained features and adding value through consensus clustering and interpretable descriptions.
