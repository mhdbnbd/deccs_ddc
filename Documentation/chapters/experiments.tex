\section{Experimental Setup}

All experiments were conducted using the pipeline described in Chapter~3. We evaluate four modes---\texttt{kmeans}, \texttt{deccs}, \texttt{ddc}, and \texttt{ddeccs}---on two datasets: AwA2 (50 classes, 85 attributes, 29857 training instances) and aPY (evaluated with both 32 classes and the DDC-matching 15-class subset). Following DDC's protocol, each experiment is repeated 10 times with different random seeds, and we report mean $\pm$ standard deviation.

Experiments were run on an NVIDIA GeForce RTX 4090 GPU. Feature extraction takes approximately 90 seconds per dataset; subsequent clustering and evaluation take seconds for K-means and 5--20 minutes for DECCS consensus (depending on dataset size). Features are cached to disk after the first extraction, so repeated runs skip this step entirely.

Several observations emerge from this comparison. First, our K-means baseline outperforms DDC's K-means baseline on both datasets (+12.7 NMI points on AwA, +2.2 on aPY). However, this comparison is not strictly apples-to-apples: DDC uses K=40 clusters on the original AwA dataset (19832 instances), while we use K=50 on AwA2 (29857 instances). Higher K generally yields higher NMI, and the different datasets contain different images. The aPY comparison is closer (both use K=15), but DDC uses AwA1-era preprocessing. Second, DDC's trained encoder network achieves higher NMI than our frozen-features approach, particularly on aPY (+19.8 points). This gap reflects the benefit of DDC's end-to-end training, which refines features specifically for the clustering task using pairwise constraints derived from semantic attributes. Third, we note that DDC uses the original AwA1 dataset while we use the replacement AwA2, making direct numerical comparison approximate.

\paragraph{Note on DDC's tag annotation ratio ($r\%$).} DDC's results in Table~\ref{tab:ddc_comparison} are reported at tag annotation ratio $r = 50\%$, meaning half of each instance's attributes are randomly masked during training. This masking is part of DDC's \emph{training procedure}---it simulates incomplete annotation and creates diversity in the pairwise constraint generation. At lower masking rates ($r = 10\%$), DDC reports lower performance (NMI = 75.62 on AwA). DDC does not report results at $r = 0\%$ (no masking), so we cannot determine how much the masking itself contributes to their performance. Our pipeline does not use masking because we do not train with pairwise constraints; we use full (unmasked) attributes only for the post-hoc ILP description step, which is comparable to DDC's ILP step, which computes cluster-level tag statistics from the attribute matrix.

\section{ILP Cluster Descriptions}
\label{sec:ilp_results}

A key contribution of our approach is the generation of interpretable cluster descriptions. Our pipeline produces two types of description output per experiment, which serve different purposes:

The \texttt{cluster\_descriptions.json} file contains a simple summary: for each cluster, the top-5 attributes ranked by their mean value within that cluster. This is a naive ranking that does not consider whether an attribute is unique to the cluster---for instance, ``Occluded'' may rank highly in many clusters because it is common across the entire dataset.

The \texttt{ilp\_descriptions.json} file contains the ILP-optimized descriptions (DDC Eq.\ 2--4). The ILP selects attributes that are both prevalent within the cluster (coverage constraint) and distinctive to that cluster (orthogonality constraint). This is why the two files show different attributes: the ILP may skip ``Occluded'' because it appears everywhere and instead select ``Pedal'' and ``Handlebars'' which uniquely identify the bicycle cluster. We focus on the ILP descriptions as the primary interpretability output.

\section{Visualizations}

Our four experimental modes produce only \emph{two} distinct sets of cluster assignments per dataset, not four. This is because the ILP description module (used in \texttt{ddc} and \texttt{ddeccs} modes) operates entirely post-hoc: it takes the cluster assignments as input and generates textual descriptions, but never modifies which instances belong to which cluster. Consequently, \texttt{ddc} produces the same cluster assignments as \texttt{kmeans} (both use K-means for clustering), and \texttt{ddeccs} produces the same assignments as \texttt{deccs} (both use DECCS consensus). The only difference between paired modes is the presence of ILP-generated descriptions, which appear in the JSON output files but do not affect visualizations.

The two genuinely distinct visualizations per dataset---K-means versus DECCS consensus---illustrate how ensemble consensus changes cluster boundaries. On AwA2, the differences are subtle because both methods achieve similar NMI. On aPY, the differences are more pronounced, reflecting DECCS's 14.4\% NMI improvement.

\section{The Path to the Current Approach: Early Experiments and Lessons Learned}
\label{sec:early_experiments}

Before arriving at the current frozen-features pipeline, we explored several alternative approaches that provided important insights into the challenges of deep descriptive clustering. We document these experiments transparently, as the negative results informed our final design decisions.

\subsection{Autoencoder-Based Approach Initial Attempt}

Our initial design followed the DECCS architecture more closely: a convolutional autoencoder with a tag prediction branch, trained with a combined loss $\mathcal{L} = \mathcal{L}_{recon} + \lambda_{tag} \cdot \mathcal{L}_{tag} + \lambda_{cons} \cdot \mathcal{L}_{consensus}$. Three encoder architectures were tested: a small 4-layer CNN ($128 \times 128$), a larger 5-layer CNN with BatchNorm ($224 \times 224$), and a frozen ResNet-101 backbone with a trainable projector.

On a 160-image sample, this approach achieved NMI = 0.642, which appeared promising. However, full-scale evaluation on 37,322 images produced NMI = 0.012---essentially random clustering. Extensive debugging revealed multiple contributing factors: DBSCAN in the ensemble was labeling all points as noise (corrupting the consensus matrix), visualization metrics were being computed on different label sets than training, and the reconstruction loss on ImageNet-normalized images was dominating the gradient signal due to a normalization mismatch between encoder input and decoder output.

\subsection{DDC-Style MLP Training Attempt}

After identifying the reconstruction loss as a fundamental obstacle, we implemented DDC's actual training objectives: mutual information maximization combined with pairwise constraints from semantic tags, using a 3-layer MLP (2048 $\to$ 1200 $\to$ 1200 $\to$ K) on top of frozen ResNet features. This approach faced a critical scaling challenge: on the full dataset, the MI loss dominated from the first epoch (reaching $L_{MI} = -0.92$ after a single epoch of 93 mini-batches), locking the model into approximately 24 coarse clusters before the pairwise constraints could influence the structure.

Several fixes were attempted. Per-instance tag masking ($r = 0.5$), which randomly zeros 50\% of each instance's attributes during training, proved essential for making pairwise constraints meaningful---without it, all instances of the same class had identical attribute vectors, causing the pairwise selection to ignore tag information entirely. This fix improved results on a 1,600-sample subset (NMI = 0.124, 30 active clusters, first positive silhouette score), but the improvement did not transfer to the full dataset. A proper ILP solver (PuLP CBC) was also implemented to replace an initial greedy heuristic that selected 75 of 85 tags (providing almost no filtering), but this too did not resolve the MI dominance issue at scale.

\subsection{The K-Means Baseline Finding}

The decisive moment came when we evaluated K-means directly on the raw 2048-dimensional ResNet features: NMI = 0.872 on AwA2. This was higher than any result achieved by the trained MLP, demonstrating that the pretrained features already contained sufficient class-discriminative information and that our MLP training was actively destroying this information rather than refining it. This finding redirected the thesis toward the current approach: leveraging the strong pretrained features and adding value through consensus clustering and interpretable descriptions.

\section{Summary of Results}

The experimental results support two main conclusions. First, DECCS consensus clustering provides significant and consistent improvements on challenging datasets: +14.4\% NMI on aPY (32 classes), with dramatically reduced variance. On AwA2, where ResNet features already provide near-perfect class separation, consensus matches K-means performance while providing improved accuracy and stability. Second, the ILP-based description generation produces meaningful, orthogonal cluster explanations that align with human intuition, with Tag Coverage values ranging from 0.18 to 0.83 and Inverse Tag Frequency up to 3.47, indicating distinctive descriptions for each cluster.
