\section{Overview}

This chapter presents the methodology of our integrated approach, which we term DDECCS (Deep Descriptive Clustering with Consensus Representations). The approach combines pretrained deep feature extraction with consensus-based ensemble clustering and interpretable cluster descriptions via Integer Linear Programming (ILP). Our pipeline operates in four sequential stages: feature extraction, clustering, description generation, and evaluation.

Unlike approaches that jointly train feature representations and clustering objectives---which can suffer from loss balancing instability as we discuss in Section~\ref{sec:design_rationale}---our method decouples feature extraction from clustering. This design choice leverages the strong visual representations already learned by modern pretrained networks while allowing each downstream component (clustering, description) to operate independently and reliably.

\paragraph{Terminology.} The DDC paper uses the term ``tags'' to refer to semantic properties of instances (e.g., ``furry,'' ``has wheels''). The dataset literature (AwA2, aPY) uses ``attributes'' for the same concept. Throughout this thesis, we use ``attributes'' as the primary term and note that DDC's ``tags'' are synonymous.

\section{Pipeline Architecture}

Our pipeline consists of four stages, illustrated in Figure~\ref{fig:pipeline}:

\begin{enumerate}
    \item \textbf{Feature Extraction}: Frozen ResNet-101 pretrained on ImageNet extracts 2048-dimensional feature vectors from input images.
    \item \textbf{Clustering}: Either K-means (baseline) or DECCS consensus ensemble assigns images to clusters.
    \item \textbf{Description Generation}: DDC's ILP formulation generates concise, orthogonal cluster descriptions using semantic attributes.
    \item \textbf{Evaluation}: Standard clustering metrics (NMI, ACC, ARI, Silhouette) and interpretability metrics (TC, ITF) assess the results.
\end{enumerate}

Algorithm~\ref{alg:ddeccs} summarizes the complete DDECCS pipeline.

\begin{algorithm}[H]
\caption{DDECCS Pipeline}
\label{alg:ddeccs}
\begin{algorithmic}[1]
    \Require Image dataset $\mathcal{X}$, attribute matrix $T$, number of clusters $K$, mode $\in \{\texttt{kmeans}, \texttt{deccs}, \texttt{ddc}, \texttt{ddeccs}\}$
    \Ensure Cluster assignments $\mathbf{y}$, cluster descriptions $\mathbf{D}$ (if ILP mode)

    \State \textbf{Step 1: Feature Extraction}
    \State Load frozen ResNet-101 (pretrained on ImageNet, final layer removed)
    \State $F \gets \emptyset$
    \For{each image $x_i \in \mathcal{X}$}
        \State Resize $x_i$ to $224 \times 224$, normalize with ImageNet mean/std
        \State $\mathbf{f}_i \gets \text{ResNet-101}(x_i) \in \mathbb{R}^{2048}$ \Comment{Global average pooling output}
        \State $F \gets F \cup \{\mathbf{f}_i\}$
    \EndFor
    \State Cache $F$ to disk for subsequent runs

    \State
    \State \textbf{Step 2: Clustering}
    \State Standardize $F$ to zero mean, unit variance
    \If{mode $\in \{\texttt{kmeans}, \texttt{ddc}\}$}
        \State $\mathbf{y} \gets \text{KMeans}(F, K, \text{n\_init}=10)$
    \ElsIf{mode $\in \{\texttt{deccs}, \texttt{ddeccs}\}$}
    \State $\pi_1 \gets \text{KMeans}(F, K)$
    \State $\pi_2 \gets \text{SpectralClustering}(F, K, k\text{-NN affinity}, k=15)$
    \State $F_{\text{pca}} \gets \text{PCA}(F, d=\min(256, \dim))$ \Comment{For GMM numerical stability}
    \State $\pi_3 \gets \text{GMM}(F_{\text{pca}}, K, \text{full covariance})$
    \State $\pi_4 \gets \text{Agglomerative}(F, K, \text{Ward linkage})$
    \State Build co-association matrix: $C_{ij} \gets \frac{1}{4}\sum_{m=1}^{4} \mathbb{1}[\pi_m(i) = \pi_m(j)]$
    \State Sparsify $C$ using $k$-nearest neighbors ($k=20$)
    \State $\mathbf{y} \gets \text{SpectralClustering}(C, K)$
    \EndIf

    \State
    \State \textbf{Step 3: ILP Description Generation} (only if mode $\in \{\texttt{ddc}, \texttt{ddeccs}\}$)
    \State Compute adaptive $\alpha \gets \text{clip}(\text{round}(16 \cdot \bar{d}), 2, 8)$ where $\bar{d}$ is mean attribute density
    \For{each cluster $i = 1, \ldots, K$}
        \For{each attribute $j = 1, \ldots, A$}
            \State $Q_{ij} \gets$ mean of attribute $j$ across instances in cluster $i$
        \EndFor
    \EndFor
    \State Solve ILP: minimize $\sum_{i,j} W_{ij}$ subject to:
    \State \quad Coverage: $\sum_j W_{ij} \cdot Q_{ij} \geq \alpha \quad \forall i$
    \State \quad Orthogonality: $\sum_i W_{ij} \leq \beta \quad \forall j$ (starting $\beta=1$, increment if infeasible)
    \State \quad $W_{ij} \in \{0, 1\}$
    \State $\mathbf{D} \gets \{(i, \{j : W_{ij} = 1\}) \mid \forall i\}$

    \State
    \State \textbf{Step 4: Evaluation}
    \State Compute NMI, ACC (Hungarian), ARI, Silhouette against ground-truth labels
    \State Compute TC, ITF for each cluster description in $\mathbf{D}$
\end{algorithmic}
\end{algorithm}

\section{Feature Extraction}
\label{sec:feature_extraction}

We use ResNet-101~\citep{He2016} pretrained on ImageNet~\citep{Deng2009} as our feature extractor. The final classification layer is removed, and the output of the global average pooling layer provides a 2048-dimensional feature vector for each image. All backbone weights are frozen during the entire pipeline---no fine-tuning is performed.

Images are preprocessed with standard ImageNet normalization: resized to $224 \times 224$ pixels, converted to tensors, and normalized with mean $[0.485, 0.456, 0.406]$ and standard deviation $[0.229, 0.224, 0.225]$. Features are extracted once and cached to disk for reuse across experiments.

This approach follows DDC~\citep{Zhang2021}, which also uses frozen ResNet-101 features, ensuring fair comparison. The use of pretrained features is well-established in the clustering literature---K-means on ResNet features already achieves NMI = 0.869 on AwA2, demonstrating that the backbone captures sufficient class-discriminative information without task-specific training.

\subsection{K-means Baseline}

K-means clustering~\citep{Lloyd1982} is applied directly to the 2048-dimensional ResNet features with $K$ clusters, 10 random initializations, and the Elkan algorithm for efficiency. This serves as the baseline against which consensus clustering is compared.

\subsection{DECCS Consensus Clustering}
\label{sec:deccs_consensus}

DECCS~\citep{Miklautz2022} improves clustering robustness through ensemble consensus. Our implementation constructs a consensus from four base clustering algorithms applied to standardized features:

\begin{enumerate}
    \item \textbf{K-means}: Fast centroid-based clustering (Elkan algorithm).
    \item \textbf{Spectral Clustering}: Graph-based method using a $k$-nearest-neighbor affinity matrix ($k=15$), which captures non-linear cluster structures.
    \item \textbf{Gaussian Mixture Model (GMM)}: Probabilistic clustering with full covariance. For high-dimensional features (e.g., 2048-dim), we apply PCA reduction to 256 dimensions before fitting GMM to ensure numerical stability.
    \item \textbf{Agglomerative Clustering}: Hierarchical method with Ward linkage, which minimizes within-cluster variance.
\end{enumerate}

The ensemble results are combined via a sparse co-association matrix $A$, where $A_{ij}$ counts how often instances $i$ and $j$ are assigned to the same cluster across the base clusterers, restricted to $k$-nearest neighbors for efficiency~\citep{Fred2005}. The final clustering applies Spectral Clustering to this consensus matrix.

This differs from the original DECCS, which trains an autoencoder with consensus loss during representation learning. Our post-hoc consensus approach applies the ensemble after feature extraction, trading the iterative refinement of learned representations for the reliability and reproducibility of frozen pretrained features. As our experiments demonstrate (Section~\ref{sec:results}), this trade-off is favorable: DECCS consensus provides significant improvements on datasets where individual clustering methods are less reliable (e.g., +14\% NMI on aPY).

\section{Cluster Description via Integer Linear Programming}
\label{sec:ilp}

Following DDC~\citep{Zhang2021}, we generate interpretable cluster descriptions by solving an Integer Linear Programming (ILP) problem that selects a small, orthogonal set of semantic attributes for each cluster.

\subsection{Problem Formulation}

Given cluster assignments $\{C_1, \ldots, C_K\}$ and a set of $M$ semantic attributes, we define the attribute coverage matrix $Q \in \mathbb{R}^{K \times M}$ where $Q_{ij}$ is the mean value of attribute $j$ across all instances in cluster $C_i$. The ILP solves for a binary allocation matrix $W \in \{0,1\}^{K \times M}$ where $W_{ij} = 1$ indicates that cluster $C_i$ is described by attribute $j$.

\textbf{Objective} (DDC Eq.\ 2): Minimize the total number of attributes used:
\begin{equation}
    \arg\min_W \sum_{i,j} W_{ij}
    \label{eq:ilp_obj}
\end{equation}

\textbf{Coverage constraint} (DDC Eq.\ 3): Each cluster must have sufficient attribute coverage:
\begin{equation}
    \sum_{j=1}^{M} W_{ij} \cdot Q_{ij} \geq \alpha \quad \forall i \in \{1, \ldots, K\}
    \label{eq:ilp_coverage}
\end{equation}

\textbf{Orthogonality constraint} (DDC Eq.\ 4): Each attribute should describe at most $\beta$ clusters:
\begin{equation}
    \sum_{i=1}^{K} W_{ij} \cdot Q_{ij} \leq \beta \quad \forall j \in \{1, \ldots, M\}
    \label{eq:ilp_orthogonal}
\end{equation}

The solver searches for the smallest feasible $\beta$ starting from $\beta = 1$, ensuring the tightest possible orthogonality. We use the PuLP library~\citep{Mitchell2011} with the CBC solver and a 30-second timeout per $\beta$ value.

\subsection{Adaptive $\alpha$ Parameter}

DDC uses a fixed $\alpha = 8$, which assumes dense attribute annotations. For datasets with sparse binary attributes, $\alpha = 8$ can make the ILP infeasible. We introduce an adaptive $\alpha$ that scales with the mean attribute value, and explain the issue with a concrete example.

The coverage constraint (Eq.~\ref{eq:ilp_coverage}) requires $\sum_j W_{ij} \cdot Q_{ij} \geq \alpha$ for each cluster. The $Q_{ij}$ values---the mean attribute values within each cluster---determine whether this sum can reach $\alpha$. Consider two scenarios:

On AwA2, attributes are continuous with mean value $\bar{d} \approx 0.5$. A typical cluster might have $Q$ values like: furry = 0.89, stripes = 0.95, big = 0.82, swims = 0.12, and so on. Selecting 10 attributes with average $Q$ value 0.8 gives a weighted sum of $10 \times 0.8 = 8.0$, meeting $\alpha = 8$.

On aPY, attributes are binary with mean value $\bar{d} \approx 0.087$. A typical cluster might have $Q$ values like: Wheel = 0.30, Metal = 0.25, Pedal = 0.15, Furry = 0.02, and so on---most values are small because only 8.7\% of all attribute entries are 1. Selecting 10 attributes with average $Q$ value 0.09 gives a weighted sum of only $10 \times 0.09 = 0.9$. Even selecting all 64 attributes yields at most $64 \times 0.087 = 5.6$, which is below 8. Hence $\alpha = 8$ is infeasible on aPY.

Our adaptive formula scales $\alpha$ proportionally to $\bar{d}$:

\begin{equation}
    \alpha = \text{clip}\left(\text{round}(16 \cdot \bar{d}), \; 2, \; 8\right)
    \label{eq:alpha_adaptive}
\end{equation}

where $\bar{d}$ is the mean attribute value across all instances and all attributes (i.e., the average entry in the attribute matrix). For binary attributes, this equals the fraction of non-zero entries. The constant 16 derives from DDC's setting: $\alpha / \bar{d} = 8 / 0.5 = 16$. The clip to $[2, 8]$ ensures at least two meaningful attributes per cluster description (lower bound) and compatibility with DDC's original setting (upper bound). This yields $\alpha = 8$ for AwA2 and $\alpha = 2$ for aPY.

\subsection{Interpretability Metrics}

Following DDC, we evaluate cluster descriptions using Tag Coverage (TC) and Inverse Tag Frequency (ITF):

\textbf{Tag Coverage} (DDC Eq.\ 8): The average fraction of cluster members possessing each descriptive attribute:
\begin{equation}
    TC(C_i) = \frac{1}{|D_i|} \sum_{d \in D_i} \frac{|\{(x,t) \in C_i : d \in t\}|}{|C_i|}
\end{equation}

\textbf{Inverse Tag Frequency} (DDC Eq.\ 9): Measures how unique each cluster's description is:
\begin{equation}
    ITF(C_i) = \frac{1}{|D_i|} \sum_{d \in D_i} \log \frac{K}{\sum_{j=1}^{K} |d \in D_j|}
\end{equation}

Higher TC indicates that the selected attributes are prevalent within the cluster; higher ITF indicates that the attributes are distinctive to that cluster rather than shared across many clusters.

\section{Datasets}
\label{sec:datasets}

We evaluate on two established attribute-annotated image datasets used in the DDC literature.

\subsection{Animals with Attributes 2 (AwA2)}

AwA2~\citep{Xian2018} contains 37,322 images across 50 animal classes, each annotated with 85 continuous semantic attributes (e.g., \textit{furry}, \textit{stripes}, \textit{swims}). Attributes are defined per class---all instances of a given class share the same attribute vector. We use the standard 80/20 train/test split, yielding 29857 training instances.

We note that DDC's original evaluation uses the AwA1 dataset~\citep{Lampert2013} (19832 images), which has been withdrawn due to copyright restrictions. AwA2 is the recommended drop-in replacement, sharing the same class definitions and attribute structure but containing different images. This means direct numerical comparison with DDC's reported results on AwA1 is approximate.

% Source: data/AwA2-data/Animals_with_Attributes2/

\subsection{Attribute Pascal and Yahoo (aPY)}

aPY~\citep{Farhadi2009} contains 15339 object instances across 32 classes from Pascal VOC 2008 and Yahoo images, annotated with 64 binary semantic attributes (e.g., \textit{Wheel}, \textit{Furry}, \textit{Metal}). Unlike AwA2, attributes in aPY are annotated per instance---each object has its own binary attribute vector based on its bounding box crop.

DDC reports results on a 15-class subset of aPY consisting of 5,274 instances. In our dataset, filtering to these 15 classes yields 6,487 instances; after our standard 80/20 train/test split, we cluster on 5,189 training instances. The discrepancy with DDC's reported count likely reflects differences in preprocessing or filtering criteria. Following their setup, we evaluate with both the full 32-class dataset and the 15-class subset matching DDC's configuration. The 15 classes, identified from DDC's Figure~3 ontology, are: dog, cat, monkey, zebra, bird, bag, mug, bottle, chair, car, building, bicycle, boat, sofa, and dining table.

\section{Design Rationale: Why Frozen Features Instead of End-to-End Training}
\label{sec:design_rationale}

Our initial approach attempted to replicate DDC's end-to-end training with a trainable MLP on top of frozen ResNet features, using mutual information maximization and pairwise constraints from semantic tags. This approach faced fundamental scaling challenges: on the full AwA2 dataset (29857 instances), the mutual information loss dominated from the first epoch ($L_{MI} = -0.92$ at epoch 1), locking the model into approximately 24 clusters before pairwise constraints could influence the cluster structure. Despite extensive debugging---including per-instance tag masking ($r = 0.5$), proper ILP implementation via PuLP, learning rate tuning, and mini-batch pairwise computation---the trained MLP consistently produced worse results (NMI $\approx 0.006$) than simple K-means on the raw features (NMI $= 0.869$).

This finding led us to the current architecture: rather than training features that are already excellent, we focus the thesis contribution on (1) consensus clustering for improved robustness and (2) ILP-based descriptions for interpretability. This decoupled approach achieves competitive results with DDC on AwA2 (NMI $= 0.869$ vs.\ DDC's $0.896$) while offering the additional benefit of consensus-based stability and the ability to evaluate clustering and description independently.

\section{Evaluation Metrics}

We evaluate clustering quality using four standard metrics:

\textbf{Normalized Mutual Information (NMI)}~\citep{Strehl2002} measures the agreement between predicted clusters and ground-truth classes, normalized to $[0,1]$. NMI $= 1$ indicates perfect agreement.

\textbf{Clustering Accuracy (ACC)} uses the Hungarian algorithm~\citep{Kuhn1955} to find the optimal one-to-one mapping between predicted clusters and true classes, then computes the fraction of correctly assigned instances.

\textbf{Adjusted Rand Index (ARI)}~\citep{Hubert1985} measures pairwise agreement between clusterings, adjusted for chance. ARI $= 0$ indicates random clustering; ARI $= 1$ indicates perfect agreement.

\textbf{Silhouette Score}~\citep{Rousseeuw1987} measures how similar instances are to their own cluster compared to other clusters, ranging from $-1$ to $+1$.

Following DDC's experimental protocol, all experiments are repeated 10 times with different random seeds, and we report mean $\pm$ standard deviation.
