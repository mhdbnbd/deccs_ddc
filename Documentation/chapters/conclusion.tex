\section{Summary of Contributions}

This thesis investigated the integration of Deep Descriptive Clustering (DDC) interpretability mechanisms into the DECCS consensus clustering framework, with evaluation on the AwA2 and aPY attribute-annotated image datasets. The resulting framework, DDECCS, combines pretrained ResNet-101 feature extraction, ensemble-based consensus clustering, and Integer Linear Programming for generating interpretable cluster descriptions.

The work makes three contributions. First, we demonstrate that DECCS consensus clustering provides significant improvements over K-means on challenging multi-category datasets (+14.4\% NMI on aPY), while achieving comparable performance on well-separated datasets (AwA2) with improved stability and accuracy. Second, we implement and adapt DDC's ILP-based description generation for datasets with varying attribute density, introducing an adaptive $\alpha$ parameter that ensures feasibility on sparse binary attributes. Third, we provide a thorough empirical comparison across two datasets, four clustering modes, and ten random seeds, establishing reproducible baselines for future work.

\section{Research Questions Answered}

\textbf{RQ1: How can DDC be integrated into DECCS to improve interpretability?} Through post-hoc ILP description generation that operates on DECCS consensus cluster assignments. The ILP produces concise (4--6 attributes per cluster), orthogonal ($\beta = 1$ on aPY), and semantically meaningful descriptions (average TC = 0.70 on AwA2). This integration adds interpretability without affecting clustering performance.

\textbf{RQ2: What impact does the integration have on clustering performance?} The ILP has zero impact on clustering metrics, as it operates post-hoc. DECCS consensus itself improves performance on aPY (+14.4\% NMI) and provides increased stability (near-zero standard deviation across seeds) on both datasets.

\textbf{RQ3: How does the approach perform on AwA2 and aPY?} On AwA2: NMI = 0.871, competitive with DDC's 0.896. On aPY (15-class): NMI = 0.677, below DDC's 0.863. The gap is attributable to our use of frozen features versus DDC's task-specific MLP training. Our K-means baseline outperforms DDC's baseline on both datasets, though this comparison is confounded by different K values (K=50 vs K=40 on AwA) and different datasets (AwA2 vs AwA).

\section{Lessons Learned}

\subsection{Pretrained Features Can Be Sufficient}

The most surprising finding was that K-means on raw ResNet-101 features (NMI = 0.869 on AwA2) outperformed every trained model we built. Modern pretrained backbones encode rich visual semantics that do not need further refinement for many clustering tasks. This suggests that the deep clustering literature's focus on end-to-end training may overlook simpler, more reliable approaches for datasets where pretrained features are already discriminative.

\subsection{Loss Balancing in Deep Clustering Is Fragile}

Our extensive experiments with DDC-style MLP training revealed that mutual information maximization dominates pairwise constraint losses on large datasets. A fixed loss weight $\lambda$ that works at one scale (1,600 samples) fails at another (29,857 samples) because the relative magnitudes of loss terms change with dataset size and number of mini-batches per epoch. This fragility suggests that adaptive loss balancing techniques~\citep{Crawshaw2020} may be necessary for scaling deep descriptive clustering.

\subsection{Experimental Setup Details Matter}

The difference between using all 32 aPY classes versus DDC's 15-class subset changed our NMI from 0.418 to 0.666---a 59\% improvement from correcting the experimental setup alone. Similarly, the discovery that DDC's per-instance tag masking ($r = 0.5$) was essential for pairwise constraints (without it, same-class instances had identical tags, making pairwise selection ignore tag information) highlights how crucial implementation details are for reproducing published results.

\section{Limitations}

The primary limitation is the absence of task-specific feature learning, which accounts for the performance gap with DDC, particularly on aPY. Our post-hoc ILP cannot improve clustering through iterative refinement as in DDC's original framework. Additionally, the approach requires predefined semantic attributes, limiting applicability to domains with existing attribute vocabularies. Finally, our DECCS implementation applies consensus post-hoc on frozen features rather than during training, which differs from the original DECCS algorithm.

\section{Future Work}

\subsection{Resolving the MLP Training Challenge}

The core technical challenge---making DDC's MLP training work at scale---remains open. Potential directions include adaptive loss balancing (e.g., GradNorm~\citep{Chen2018GradNorm}), learning rate warmup schedules that allow pairwise constraints to establish structure before MI commits to cluster assignments, and curriculum-based training that gradually increases dataset size.

\subsection{End-to-End Consensus with Descriptions}

Integrating the ILP into a training loop where descriptions inform consensus construction could create the iterative refinement cycle that our post-hoc approach lacks. This would require differentiable approximations of the ILP, such as continuous relaxations of the binary allocation matrix.

\subsection{Extension to Other Domains}

The DDECCS framework is applicable to any domain with pretrained feature extractors and semantic attribute annotations. Natural extensions include medical image clustering with clinical attributes, scientific image analysis with domain-specific descriptors, and text clustering with topic-based attributes.

\section{Final Remarks}

This thesis demonstrates that interpretable clustering does not require sacrificing performance. By combining the strengths of consensus clustering (robustness, stability) with ILP-based descriptions (conciseness, orthogonality), the DDECCS framework provides both accurate clusters and human-understandable explanations. The journey from initial autoencoder experiments through MLP training challenges to the final frozen-features pipeline illustrates that in deep learning research, the path to a working solution is rarely linear, and documenting what did not work is as valuable as reporting what did.
